A simple program that reads from a CSV file synthetic trip data in Brussels generated by the MobilityDB-BerlinMOD generator, https://github.com/MobilityDB/MobilityDB-BerlinMOD disassembles the trips into individual observations, and write them in a CSV file named "berlinmod_instants.csv" ordered by timestamp.

The input file is

  • berlinmod_trips.csv: 55 trips from 5 cars during 4 days obtained from the generator at scale factor 0.005. The input file has been generated with the following SQL command on the database containing the generated data
    COPY (SELECT tripid, vehid, day, seqno, ashexewkb(trip) AS trip FROM trips WHERE vehid < 6 ORDER BY tripid) TO '/home/user/src/berlinmod_trips.csv' CSV HEADER;
    ORDER
    #define ORDER
    Symbolic constants for sets and for normalizing spans.
    Definition: temporal.h:76
    In the above file, the coordinates are given in the 3857 coordinate system, https://epsg.io/3857 and the timestamps are given in the Europe/Brussels time zone. This simple program does not cope with erroneous inputs, such as missing fields or invalid values.

The program can be build as follows

gcc -Wall -g -I/usr/local/include -o 05_berlinmod_disassemble 05_berlinmod_disassemble.c -L/usr/local/lib -lmeos
