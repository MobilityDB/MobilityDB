120/*****************************************************************************
121 * M2 — rotational case (2D only)
122 *
123 * Generalises the M1 API to allow non-zero rotation between pose1 and pose2.
124 * The moving body's edge endpoints in the local frame (`p_a_local`,
125 * `p_b_local`) trace circular arcs in world space under the interpolated
126 * pose. Pure-translation input (theta1 == theta2) delegates to the M1
127 * fast path internally.
128 *
129 * 3D pose input (Z-flag set) is rejected with -1; M2 is 2D only, matching
130 * the design constraint that the trgeometry / cbuffer surfaces use
131 * 2D-only kernels (PostGIS curve arithmetic and GEOS topology operate
132 * in 2D).
133 *****************************************************************************/
