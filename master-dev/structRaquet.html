Detailed Description

Structure to represent a Raquet raster tile.

Variable-length (varlena) value. The field order is chosen so that the 8-byte members are naturally aligned and no internal padding is needed, hence the pixel payload starts at a fixed offset. The pixels are stored row-major, tightly packed, width * height * raquet_pixtype_size(pixtype) bytes; the tile geotransform is fully determined by quadbin.
