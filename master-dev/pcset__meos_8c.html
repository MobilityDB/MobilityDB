Detailed Description

Set types over pgpointcloud pcpoint / pcpatch base values.

Mirrors the structure of geoset_meos.c and cbufferset_meos.c. Both set types enforce pcid uniformity across their elements — values with different schemas cannot coexist because their byte layouts are incomparable at the MEOS layer (dimension-level access needs the schema XML from pointcloud_formats, unavailable here). The constructor-path check lives in set_make_exp in meos/src/temporal/set.c; the set-operation path uses ensure_valid_pcpointset_pcpoint / ensure_valid_pcpatchset_pcpatch implemented in pcpoint.c / pcpatch.c.

These sets carry no bounding box — pointcloudset_type(settype) makes set_bbox_size return 0. TPCBox is the separate type that carries pointcloud spatial bounds.
