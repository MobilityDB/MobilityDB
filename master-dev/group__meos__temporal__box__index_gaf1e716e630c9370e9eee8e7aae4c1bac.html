void rtree_insert_temporal_split ( RTree *  rtree,
const Temporal *  temp,
int  id,
int  maxboxes 
)

Insert a temporal value into the RTree index as several tight per-segment bounding boxes sharing the same id (multi-entry indexing)

The temporal value is decomposed into at most maxboxes tight per-segment bounding boxes, all inserted under id. This yields a more selective index than rtree_insert_temporal for wiggly or high-extent temporal values, at the cost of more leaf entries. The temporal type must be compatible with the RTree's bounding box type: temporal alphas (tbool, ttext) require a span-based RTree, temporal numbers (tint, tfloat) require a TBox-based RTree, and spatiotemporal types (tgeompoint, tgeogpoint) require an STBox-based RTree. When maxboxes <= 1, the temporal value is an instant, or the spatial type has no per-segment STBox decomposition, the behaviour is identical to rtree_insert_temporal (a single minimum bounding box). The tree, split and insertion algorithms are unchanged; this only inserts several boxes under the same id, which the parallel ids/boxes leaf layout already supports. Search results may contain the same id several times; use rtree_search_temporal_dedup to collapse them.

Parameters
[in]rtreeThe RTree previously initialized
[in]tempThe temporal value to be inserted
[in]idThe id of the temporal value being inserted, shared by every box produced for temp
[in]maxboxesMaximum number of bounding boxes produced for temp; values <= 1 degenerate to the single minimum bounding box
